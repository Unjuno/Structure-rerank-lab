# External benchmark plan

Goal: compare structure-aware reranking against standard retrieval datasets, not only hand-written samples.

## First external datasets

Use small BEIR-style datasets first:

| dataset | reason |
|---|---|
| `arguana` | argument retrieval; closest to structure and reasoning search |
| `scifact` | scientific claim/evidence retrieval |
| `nfcorpus` | medical/nutrition retrieval with different domain language |

## Format target

Convert each dataset into the lab's existing JSONL format:

- `posts.jsonl`
- `queries.jsonl`
- `structures.jsonl`
- `judgments.jsonl`

Then run the same evaluation path:

```bash
python -m structure_rerank.vector_experiment \
  --posts results/beir/<dataset>/posts.jsonl \
  --queries results/beir/<dataset>/queries.jsonl \
  --structures results/beir/<dataset>/structures.jsonl \
  --output results/beir/<dataset>/vector_results.jsonl

python -m structure_rerank.mode_feedback \
  --results results/beir/<dataset>/vector_results.jsonl \
  --judgments results/beir/<dataset>/judgments.jsonl \
  --output-md results/beir/<dataset>/vector_feedback.md \
  --output-json results/beir/<dataset>/vector_feedback.json \
  --baseline-mode vector_baseline \
  --candidate-mode vector_structure_rerank
```

## Important limitation

BEIR datasets do not provide our five structure labels directly. Initial `structures.jsonl` will therefore be generated by weak heuristic extraction. That means the first external benchmark measures:

```text
TF-IDF vector baseline + weak heuristic structure rerank
```

It does not yet measure human-quality or LLM-quality structure extraction.

## Decision rule

- If structure rerank improves on `arguana`, the argument-search hypothesis gets support.
- If it fails on `scifact` or `nfcorpus`, inspect whether the heuristic structure extractor is too weak before rejecting the reranking idea.
- Do not average across datasets blindly. Report per-dataset values.
